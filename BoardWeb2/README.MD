aop 부분에 대한 언급을 넣을 예정.

어노테이션으로 aop를 처리하면 오류가 굉장히 많이 나고 대부분 충돌에서 생긴 문제가 발생한다.
아마 component-scan, aspectj 등으로 xml에서 거의 전영역을 처리하기 때문에 bean 객체가 중구난방으로 생성되고 실제로 Client에서 원하는 처리를 하기보다 bean객체로 올라온 모든 것을 다 처리하려고 해서 문제가 발생하는 것 같다.

충돌이라기 보다는 모든 객체를 다 처리하려고 하다가 처리하지 못한 객체가 존재해서 컴파일러에서 Exception으로 여겨서 예외처리를 하고 그로 인해 에러가 Console에 찍힌다.
위치를 명확히 해주면 에러가 줄어들긴 하나, xml에서 구체적으로 지정해주어야 하고 정작 xml에서 component-scan을 두개 넣어서 구체적으로 지정하려고 하면 pointcut이 정상적으로 작동을 하지 않아서 해결하기 복잡하다.

AOP를 XML에서 전부 처리하기에는 XML이 길어져서 어노테이션을 이용하는지 모르지만, AOP 영역은 XML에서 처리하는게 더 합리적이라는 생각이 든다.
